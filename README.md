# QA Testing Project – Plataforma Educativa

## 📌 Descripción
Este proyecto consiste en la validación de una plataforma educativa utilizada para la gestión de alumnos, cursos y sesiones en línea, con el objetivo de asegurar su correcto funcionamiento antes y durante su operación.

## 🎯 Objetivo
Garantizar la calidad del sistema mediante la ejecución de pruebas funcionales y validación de procesos críticos.

## 🔍 Alcance
- Pruebas funcionales de módulos académicos
- Validación de flujos de inscripción
- Pruebas en ambientes QA, Staging y Producción
- Validación de datos en base de datos
- Testing en múltiples navegadores

## 🧪 Actividades realizadas
- Ejecución de pruebas manuales sobre la plataforma web
- Validación de procesos de inscripción de alumnos
- Identificación y documentación de errores
- Seguimiento de incidencias hasta su resolución
- Validación de datos mediante consultas en MySQL
- Pruebas en navegadores (Chrome, Firefox, Edge, Safari)

## 🛠 Herramientas utilizadas
- Postman
- MySQL
- Trello
- Navegadores Web

## 📊 Resultados
Se detectaron fallos en procesos críticos relacionados con la inscripción de alumnos y funcionamiento de módulos, los cuales fueron documentados y corregidos, mejorando la estabilidad del sistema.

Contexto del proyecto

Se realizó la validación de una plataforma educativa utilizada para la gestión de alumnos, cursos y sesiones en línea. El objetivo fue asegurar la correcta operación de los procesos académicos y administrativos dentro del sistema.

Análisis

Se ejecutaron pruebas funcionales sobre módulos clave como inscripción de alumnos, seguimiento de cursos y gestión de información académica. También se realizaron pruebas en ambientes QA, Staging y Producción para validar el comportamiento del sistema en diferentes etapas.

Se detectaron errores en procesos de inscripción, inconsistencias en la visualización de datos y fallos en la actualización de información. Para validar la integridad de los datos, se realizaron consultas en MySQL.

Asimismo, se dio seguimiento a incidencias reportadas, asegurando su documentación y resolución.

Conclusiones principales

El proyecto permitió mejorar la estabilidad de la plataforma educativa al identificar y corregir fallos en procesos críticos. La validación de datos y el seguimiento de incidencias contribuyeron a optimizar la experiencia de los usuarios y asegurar la confiabilidad del sistema.
